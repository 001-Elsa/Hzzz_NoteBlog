### 4.1 网络层概述
#### 4.1.0
网络层——服务&协议：
		发送端、接收端、段
		网络层协议存在于**每一个**主机和路由器
		路由器
	关键功能——转发分组（输入->输出）+路由选择（决定分组从**发送主机**到**目标接收主机**的路径：路由选择协议&算法）
#### 4.1.1数据平面
核心：**转发**
	传统方式：基于目标地址 +转发表（路由表）
		核心原则
		- 只查看分组的 **目标 IP 地址**
		- 使用 **最长前缀匹配**
		- 查表结果：`(下一跳, 输出端口)`
	SDN 方式：基于多个字段+流表
		核心变化
		- 不只看目标地址    
		- 匹配 **多个字段**（报头中的任意组合）	    
		- 转发决策来自 **集中控制器** 下发的 **流表**    
		- 数据平面只负责：**匹配 + 动作**

### 4.1.2控制平面
= 从**源到目的**的路径规划过程（路由）
	2种控制平面方法:
		• 传统路由算法： 在每一个路由器中的单独路由器算法元件，在控制平面进行交互。
			**含义：**
			- 每个路由器**自己运行**路由算法（如OSPF、BGP）	    
			- 路由器之间通过**交换路由消息**来学习网络拓扑
			- 每个路由器**独立计算**自己的转发表
		• Software-defined networking(SDN):在远程服务器实现

#### 最长前缀匹配
为什么需要最长前缀匹配？
		 1、路由器的转发表不可能为每一个IP地址都单独设置一个表项（那样表会太大）。
		2、因此采用**聚合**的方式：一个表项覆盖一个地址范围（一个前缀）。
		3、但聚合会导致**重叠**：一个IP地址可能同时匹配多个表项。
	解决方法：
		当多个表项都匹配时，选择**网络前缀最长（子网掩码最精确）**的那个表项进行转发。

### 4.3 网际协议 IP
#### 数据报格式
IP数据包首部通常是20字节（若包含可选项长度更长≤4字节，即32位）
**IP 协议版本号 (Version)**：标识 IP 协议的版本（如 IPv4 或 IPv6）
**头部长度 (Header Length)**：标识 IP 首部的具体长度（以字节为单位）
**服务类型 (Type of Service)**：用于指定数据报的优先级或服务质量需求 
**数据报总长度 (Total Length)**：整个 IP 数据报（首部 + 数据）的长度，最大为 64K 字节 。
**标识 (Identifier)、标志 (Flags)、片偏移 (Fragment Offset)**：这三个字段共同用于 IP 分片（Fragmentation）与重组 。
     当数据报长度超过链路层最大传输单元（MTU，通常为 1500 字节或更小）时，路由器会将数据报分片 。
        同一数据报的所有分片拥有相同的标识（ID），通过不同的偏移量来标识其在原数据报中的位置 。
**寿命 (Time to Live, TTL)**：数据报在网络中允许经过的最大路由器跳数，每经过一个路由器该值减 1，减至 0 时数据报将被丢弃，防止环路 。
**上层协议 (Upper Layer Protocol)**：指示数据部分承载的是哪种协议（如 TCP 或 UDP） 。
**首部检验和 (Header Checksum)**：用于检验 IP 首部在传输过程中是否出现错误 。
**源 IP 地址 (Source IP Address)**：发送方的 32 位 IP 地址 。
**目的 IP 地址 (Destination IP Address)**：接收方的 32 位 IP 地址 。
**可选项 (Options)**：可选字段，例如记录时间戳或指定经过的路由器列表（使用较少）

#### 分片
##### 为什么需要分片？
**链路层MTU限制**：不同类型的链路层（如以太网、WiFi等）具有不同的**最大传输单元（MTU）**，即链路层帧所能携带的最大数据长度 。
**数据报处理**：当一个IP数据报的大小超过路径上某个网络的MTU时，它就必须被分割成若干个更小的数据报（即“分片”），以便在这些受限的链路上通过 。
##### IP分片机制
大的IP数据报在网络上被分片：
一个数据报被分割成若干个小的数据报
	相同的ID，不同的偏移量
	最后一个分片标记为0
“重组”只在最终的目标主机进行
IP头位的信息被用于标识，排序相关分片
举例：
每次分了新的片都要加20字节的头部

##### 子网
IP地址：低位bits(主机部分)+高位Bits(子网部分)
WHAT：在一个子网内，所有接口的 IP 地址的高位部分（前缀）必须相同。在物理上，这些接口通过路由器连接在一起，并且无需经过其他路由器即可直接相互通信。
子网个数：要判断一个子网，将每一个接口从主机或者路由器上分开，构成了一个个网络的孤岛；每一个孤岛（网络）都是一个子网（被称之为 subnet）
##### 特殊IP地址-约定
子网部分: 全为0---本网络
主机部分: 全为0---本主机；全为1--广播地址，这个网络的所有主机
特殊IP地址：127.0.0.1、0.0.0.0

#### NAT：网络地址转换
##### why
NAT 的出现主要是为了缓解 IPv4 地址空间枯竭的问题，同时也带来了网络管理的灵活性和安全性：
- **节约公网 IP 地址**：本地局域网（LAN）中的所有设备可以共享一个公网 IPv4 地址。这意味着即使局域网内有几十台设备，也只需要向 ISP 申请一个 IP 地址。
- **内网灵活性**：如果您更换了 ISP，或者需要在局域网内重新编址，无需通知外部世界，也无需修改内部设备的配置    
- **安全性（隐蔽性）**：局域网内部的设备对于外部网络是不可见的，因为外部看到的只有 NAT 路由器的公网 IP，这起到了一定的保护作用。
##### 实现

S2、NAT 转换与记录（转换）：把**源地址**和**端口号**替换为公网IP和新端口号（必须同时修改IP和端口号->端口号用于区分具体哪台设备）

##### 隧道与封装
IPv4 过渡到 IPv6
由于互联网庞大，无法要求全球路由器同时升级。为了实现过渡，IPv6 采用了**隧道 (Tunneling)** 技术 。
- **核心原理**：当 IPv6 数据报需要穿过一个只支持 IPv4 的网络时，路由器会将整个 IPv6 数据报封装在一个 IPv4 数据报的载荷（Payload）中 。  
- **物理视图**：在 IPv4 网络中，这些被“伪装”的数据报就像普通 IPv4 数据包一样被转发 。    
- **逻辑视图**：在两个 IPv6 路由器眼中，它们之间就像有一条直接连接的隧道，可以直接通信 。
